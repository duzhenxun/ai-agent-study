# AI Agent Study - Go 实现版本

> **从简单到复杂：12个演进阶段的智能体系统完整实现**

这是一个 AI 智能体系统的完整演进项目，从最基础的循环（S01）到企业级的多智能体协作系统（S12），展示了如何逐步构建一个功能完整的 AI 代理。

## 🎯 项目概述

本项目通过 12 个渐进式版本，展示了 AI 智能体系统的完整演进路径：

- **S01-S06**: 基础框架和核心功能
- **S07-S12**: 高级协作和企业级特性

每个版本都是完整可运行的 Go 程序，展示了特定的架构模式和功能特性。

## 🏗️ 架构演进

### 第一阶段：基础代理框架 (S01-S02)

#### S01: The Agent Loop (智能体循环)

> _"一个工具 + 一个循环 = 一个智能体"_

**核心特性**：

- 基础 OpenAI API 集成
- 单一 bash 工具
- 简单的消息循环机制
- `stop_reason` 控制的退出条件

**运行方式**：

```bash
cd s01 && go run main.go
```

#### S02: Tool Use (工具使用)

> _"加一个工具, 只加一个 handler"_

**核心特性**：

- 工具分发系统
- 路径安全检查 (`safePath`)
- 4个基础工具：`bash`, `read_file`, `write_file`, `edit_file`
- 工具注册映射表

**运行方式**：

```bash
cd s02 && go run main.go
```

### 第二阶段：任务管理革新 (S03-S04)

#### S03: TodoWrite (待办写入)

> _"没有计划的 agent 走哪算哪"_

**核心特性**：

- TodoManager 状态管理
- 任务状态：`pending` → `in_progress` → `completed`
- Nag reminder 机制（3轮后提醒）
- 单一任务进行限制

**运行方式**：

```bash
cd s03 && go run main.go
```

#### S04: Subagents (子智能体)

> _"大任务拆小, 每个小任务干净的上下文"_

**核心特性**：

- 上下文隔离的子智能体
- 独立的消息循环
- 摘要返回机制
- 父子智能体通信

**运行方式**：

```bash
cd s04 && go run main.go
```

### 第三阶段：功能扩展探索 (S05-S06)

#### S05: Skills (技能加载)

> _"用到什么知识, 临时加载什么知识"_

**核心特性**：

- 动态技能加载系统
- 两层知识注入（系统提示 + 按需加载）
- 技能元数据管理
- `skills/*/SKILL.md` 文件结构

**运行方式**：

```bash
cd s05 && go run main.go
```

#### S06: Context Compact (上下文压缩)

> _"上下文总会满, 要有办法腾地方"_

**核心特性**：

- 三层压缩策略
- Micro-compact：旧结果替换为占位符
- Auto-compact：token阈值触发摘要
- Manual-compact：按需压缩工具
- 转录持久化到 `.transcripts/`

**运行方式**：

```bash
cd s06 && go run main.go
```

### 第四阶段：高级任务处理 (S07-S08)

#### S07: Task System (任务系统)

> _"大目标要拆成小任务, 排好序, 记在磁盘上"_

**核心特性**：

- 持久化任务图（JSON文件）
- 依赖关系管理：`blockedBy` 和 `blocks`
- 任务状态机：`pending` → `in_progress` → `completed`
- 自动依赖解除机制

**运行方式**：

```bash
cd s07 && go run main.go
```

#### S08: Background Tasks (后台任务)

> _"慢操作丢后台, agent 继续想下一步"_

**核心特性**：

- 后台异步任务执行
- 通知队列系统
- 300秒超时控制
- 并发安全的任务管理

**运行方式**：

```bash
cd s08 && go run main.go
```

### 第五阶段：团队协作系统 (S09-S12)

#### S09: Agent Teams (智能体团队)

> _"任务太大一个人干不完, 要能分给队友"_

**核心特性**：

- 多智能体团队管理
- JSONL 收件箱通信
- 团队配置持久化
- 独立的队友线程

**运行方式**：

```bash
cd s09 && go run main.go
```

#### S10: Team Protocols (团队协议)

> _"队友之间要有统一的沟通规矩"_

**核心特性**：

- 请求-响应协议系统
- 关机握手控制
- 计划审批流程
- FSM 状态管理

**运行方式**：

```bash
cd s10 && go run main.go
```

#### S11: Autonomous Agents (自治智能体)

> _"队友自己看看板, 有活就认领"_

**核心特性**：

- 自主任务认领
- 工作/空闲双阶段循环
- 身份重注入机制
- 60秒空闲超时

**运行方式**：

```bash
cd s11 && go run main.go
```

#### S12: Worktree + Task Isolation (工作树任务隔离)

> _"各干各的目录, 互不干扰"_

**核心特性**：

- Git worktree 隔离执行
- 任务-worktree 绑定
- 事件流日志系统
- 并行执行通道

**运行方式**：

```bash
cd s12 && go run main.go
```

## 🛠️ 环境配置

### 前置要求

- Go 1.19+
- OpenAI API 密钥
- Git（用于 S12 worktree 功能）

### 环境变量设置

1. 复制示例配置文件：

```bash
cp .env.example .env
```

2. 编辑 `.env` 文件：

```env
MODEL_ID=gpt-4
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_API_KEY=your_api_key_here
OPENAI_AUTH_HEADER=Authorization
```

### 配置特性

- **向上搜索**：自动从当前目录向上查找 `.env` 文件
- **引号处理**：自动移除环境变量值中的引号
- **容错机制**：未找到 `.env` 文件时使用系统环境变量

# 🎯 应用场景指南

### 🛠️ 开发和测试环境

#### 环境配置

所有版本（S01-S12）现在都支持统一的环境变量配置管理：

**配置文件设置**：

```bash
# 1. 复制示例配置文件
cp .env.example .env

# 2. 编辑配置文件，填入你的实际 API 密钥
# 编辑 .env 文件，修改以下配置：
MODEL_ID=gpt-4
OPENAI_BASE_URL=https://api.openai.com/v1/chat/completions
OPENAI_API_KEY=your-actual-api-key-here
OPENAI_AUTH_HEADER=Authorization

# 3. 运行任意版本
cd s01  # 或 s02, s03, ..., s12
go run main.go
```

**配置文件说明**：

- `.env.example` - 示例配置文件模板
- `.env` - 实际配置文件（需要手动创建）
- 支持注释行（以 # 开头）
- 支持空行跳过
- 只在环境变量不存在时才设置，避免覆盖系统变量
