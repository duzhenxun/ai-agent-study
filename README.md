## 🚀 功能演进历程

### 第一阶段：基础代理框架 (S01-S02)

**S01 - 起步版本**

- 核心功能：最基础的 OpenAI API 集成
- 系统提示：`"You are a coding agent at %s. Use bash to solve tasks. Act, don't explain."`
- 技术特点：单一 bash 工具，简单的对话循环
- 学习价值：理解 AI 代理的基本架构

**S02 - 安全增强**

- 核心功能：添加安全路径检查和基础工具框架
- 系统提示：`"You are a coding agent at %s. Use tools to solve tasks. Act, don't explain."`
- 技术特点：引入 `safePath` 函数，防止路径遍历攻击
- 重要改进：工具调用系统标准化

### 第二阶段：任务管理革新 (S03-S04)

**S03 - Todo 管理系统**

- 核心功能：引入 Todo 工具，支持多步骤任务规划
- 系统提示：`"You are a coding agent at %s. Use the todo tool to plan multi-step tasks. Mark in_progress before starting, completed when done. Prefer tools over prose."`
- 技术特点：任务状态管理（pending/in_progress/completed）
- 重要改进：结构化任务处理流程

**S04 - 子代理委托**

- 核心功能：任务分解和子代理委托系统
- 系统提示：引入子代理系统提示
- 技术特点：支持复杂任务的自动分解
- 重要改进：分布式任务处理能力

### 第三阶段：功能扩展探索 (S05-S06)

**S05 - 技能加载系统**

- 核心功能：可扩展的技能知识库
- 技术特点：动态技能加载，模块化知识管理
- 重要改进：支持领域专业知识集成
- 创新点：技能元数据管理系统

**S06 - 转录管理**

- 核心功能：会话历史记录和管理
- 技术特点：转录持久化，历史查询
- 重要改进：会话连续性和上下文保持
- 实用价值：长期项目跟踪

### 第四阶段：高级任务处理 (S07-S08)

**S07 - 完整任务管理系统**

- 核心功能：企业级任务管理
- 系统提示：`"You are a coding agent at %s. Use task tools to plan and track work."`
- 技术特点：任务生命周期管理，状态跟踪
- 重要改进：完整的任务工作流

**S08 - 后台任务执行**

- 核心功能：长时间运行任务支持
- 系统提示：`"You are a coding agent at %s. Use background_run for long-running commands."`

- 重要改进：非阻塞操作支持

## 🎯 应用场景指南

### 🛠️ 开发和测试环境

#### 环境配置

所有版本（S01-S12）现在都支持统一的环境变量配置管理：

**配置文件设置**：

```bash
# 1. 复制示例配置文件
cp .env.example .env

# 2. 编辑配置文件，填入你的实际 API 密钥
# 编辑 .env 文件，修改以下配置：
MODEL_ID=gpt-4
OPENAI_BASE_URL=https://api.openai.com/v1/chat/completions
OPENAI_API_KEY=your-actual-api-key-here
OPENAI_AUTH_HEADER=Authorization

# 3. 运行任意版本
cd s01  # 或 s02, s03, ..., s12
go run main.go
```

**配置文件说明**：

- `.env.example` - 示例配置文件模板
- `.env` - 实际配置文件（需要手动创建）
- 支持注释行（以 # 开头）
- 支持空行跳过
- 只在环境变量不存在时才设置，避免覆盖系统变量
